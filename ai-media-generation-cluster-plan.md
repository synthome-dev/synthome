# Content Cluster: AI-Powered Media Generation with AI Agents

## Research Summary

Based on research from industry-leading SEO resources (Semrush and Ahrefs), here's what makes an effective topic cluster for Google:

### What Google Looks For:
1. **Clear conceptual page hierarchy** - Google's official guidance
2. **Topical authority** - Comprehensive coverage of a subject demonstrates E-E-A-T (Experience, Expertise, Authoritativeness, Trustworthiness)
3. **Internal linking structure** - Helps search engines understand semantic relationships
4. **User-focused organization** - Content should be easily discoverable by both users and search engines

### Topic Cluster Components:
1. **Pillar Page** - Main hub covering the broad topic at a high level
2. **Cluster Pages** - Supporting pages diving deep into subtopics
3. **Internal Links** - Bidirectional linking between pillar and cluster pages

```
Main Pillar (Media Generation)
    ↕
Cluster Pillars (Image, Audio, Video, Agents)
    ↕
Cluster Subpages (How-to, Best Practices, Use Cases)
```

### Linking Rules
1. **Main Pillar** links to all cluster pillars
2. **Cluster Pillars** link back to main pillar and to all their subpages
3. **Subpages** link to:
   - Their cluster pillar
   - Related subpages in same cluster
   - Relevant subpages in other clusters
   - Technical documentation where appropriate

### Anchor Text Strategy
- Use descriptive, keyword-rich anchor text
- Vary anchor text naturally
- Include contextual information around links
- Example: "Learn more about [best practices for AI image generation](#)" instead of "click here"

## KEYWORD RESEARCH NOTES

### Primary Intent
Most keywords in this cluster satisfy **informational intent** with some **commercial investigation** intent (comparison pages).

### Search Volume Considerations
- Focus on keywords with 100+ monthly searches
- Include long-tail variations (lower volume, higher conversion)
- Target question-based queries for featured snippets

### Featured Snippet Opportunities
Target these question formats:
- "What is [topic]?"
- "How to [action]?"
- "Best [topic] for [use case]"
- "[Topic] vs [alternative]"
- "Why use [topic]?"

### Keyword Clustering
Group keywords by intent and SERP overlap to avoid cannibalization:
- Use Ahrefs "Traffic Share by Pages" to identify
- Create one page per keyword cluster
- Update keyword mapping document regularly

## NOTES FROM RESEARCH

### What Makes Topic Clusters Effective
- **Wikipedia model**: Every Wikipedia article is a perfect topic cluster - fully covers topics with internal linking
- **Clear hierarchy**: Google wants conceptual page hierarchy
- **E-E-A-T**: Demonstrates experience, expertise, authority, trust
- **User-first**: Content must primarily serve users, not search engines

### Common Mistakes to Avoid
- Creating too many similar pages (keyword cannibalization)
- Neglecting internal linking
- Focusing only on high-volume keywords
- Ignoring search intent
- Not grouping keywords properly
- Creating content clusters around topics too narrow

### Synthome Competitive Advantages to Highlight
- Unified API for multiple providers
- Agent-based orchestration
- Error handling and reliability
- Cost optimization
- Developer-friendly SDK
- Production-ready infrastructure
